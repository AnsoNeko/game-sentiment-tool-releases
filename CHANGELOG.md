# 更新说明

## v0.2.7 - 2026-07-16

### 新增与改进

- “指定链接采集”在批量任务部分失败后新增“重试全部失败（N）”，无需从大量原始链接中手工查找、复制和重新提交失败项。
- 每个失败结果行新增单条“重试”；成功、重复和无法识别的链接不会进入失败重试集合，也不会被重复采集。
- 批量重试继续按原链接顺序低频串行执行，只更新目标失败行；重试成功后转为“已完成”，再次失败则保留最新失败原因。
- 首轮目标项目、每链接评论上限和 Steam 语种/日期条件保存为当前页面的批次快照；用户随后修改表单时，失败重试仍沿用首轮配置，避免写入错误项目。
- 结果区实时汇总完成、失败、处理中和跳过数量；全量重新采集保留为次要操作，编辑链接或刷新/重启会清空当前页面失败集合。
- 完整保留 v0.2.6 的 Python 自动准备、稳定历史 userData 路径、SQLite v2、分平台 Cookie、安全设置和 AI 配置兼容。

### 使用方法与注意事项

1. 正常退出旧版本后运行 `GameSentimentTool-v0.2.7-win-x64-Setup.exe` 覆盖安装，无需卸载或迁移数据。
2. 指定链接批次结束后，如存在失败项，可点击底部“重试全部失败（N）”，或点击对应失败行右侧的“重试”。
3. 重试会沿用首轮目标项目和采集参数；如希望使用当前表单的新参数，请选择“重新采集全部链接”。
4. 重试状态只保留在当前页面会话，应用不会自动循环重试，也不会绕过验证码、登录风控或访问限制。
5. 安装器未进行 Authenticode 代码签名，可能触发 Windows SmartScreen；请只从本仓库 Release 下载并核对 SHA-256。

- 系统通用设置新增“舆情池数据清理”，支持按项目、平台、日期、来源归属、情绪、风险、处理状态、周报状态和关键词组合预估及清理。
- Steam 商店评价新增时间范围和五语多选，支持简体中文、英语、日语、俄语和韩语，默认仅选择简体中文。
- AI 周报严格按页面勾选的章节生成，并将全局系统 Prompt 同时应用于每批分析和最终综合。
- AI 分析上下文新增来源归属统计，可比较不同投稿人的评论量、情绪、风险、标签、点赞和回复表现。

- 更新错误提示可区分访问频率受限、网络超时、无公开 Release 和无效版本标签。

### 使用方法

1. 新用户运行 NSIS 安装程序，可选择安装目录并创建桌面或开始菜单快捷方式。
2. v0.2.1 用户可直接安装 v0.2.2。原项目、评论、周报、Cookie、安全设置和 AI 配置继续保存在当前 Windows 用户目录中，启动后会自动执行兼容迁移。

5. 在“系统通用设置 → 舆情池数据清理”中先组合筛选并点击“预估命中数”，核对数量、输入页面要求的确认文字后才可执行清理。
6. 在“周度通报自动化生成”中勾选需要的章节后再点击 AI 生成；不同来源归属的统计会随评论数据一起进入分析上下文。

### 注意事项

- 舆情池条件清理不可撤销。它只删除命中的评论及直接关联映射，保留项目、采集历史和既有周报正文；执行前仍建议导出备份。
