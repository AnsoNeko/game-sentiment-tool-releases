# 更新说明

## v0.2.3 - 2026-07-13

### 修复与改进

- 小红书主页账号 ID 不再被误识别为单笔记；主页采集会先读取最新笔记，再按配置读取评论。
- 小红书主页和评论请求加入基于完整请求路径的 `x-s`、`x-t`、`x-s-common` 动态签名。
- 抖音主页作品请求加入 `a_bogus`，评论请求加入 `X-Bogus`，并同步 Cookie 中的 `msToken` 与浏览器参数。
- 微博会根据 Cookie 类型选择桌面或移动端接口；有效的桌面 Cookie 不再因为移动端未登录状态被误报失效。
- 抖音、小红书和微博使用持久请求会话；401/明确登录失效与 403/418/429、网页验证、签名失败及平台风控分别提示。
- 开发者名称统一调整为“安索Anso”，同步应用关于页和 Windows EXE CompanyName。
- 完整保留 v0.2.2 的来源归属、条件清理、Steam 五语与日期、按勾选章节生成 AI 周报及更新检查优化。

### 使用方法

1. 下载 `GameSentimentTool-v0.2.3-win-x64-Setup.exe`，关闭正在运行的旧版本后执行安装；可直接覆盖 v0.2.1、v0.2.2 或较早的本地 v0.2.3。
2. 原有项目、评论、周报、Cookie、安全设置和 AI 配置仍保存在当前 Windows 用户目录，不需要卸载旧版或清理用户数据。
3. B站、抖音、小红书和微博采集前，请在对应网站浏览器登录账号，然后把该平台最新 Cookie 单独保存到软件设置页；不要混用不同平台 Cookie。
4. 在“游戏项目管理”配置平台账号主页；在“舆情采集与评论池 → 本地平台采集”选择项目、平台、内容数量和单内容评论数量后开始采集。
5. Steam 商店评价可按日期和简中、英语、日语、俄语、韩语多选，默认仅简体中文，多语种共用采集总量上限。
6. AI 周报生成前先在系统通用设置配置模型与系统 Prompt，再勾选本次需要生成的章节；未勾选的章节不会生成。
7. 舆情池条件清理应先预估命中数并核对筛选条件；建议在正式清除前先从导出中心备份。

### 注意事项

- 本工具只执行低频只读采集，不会绕过验证码、登录风控或访问限制，也不会自动发布、点赞、回复、私信或关注。
- 若平台提示验证码或风控，请先在浏览器中人工确认账号可正常访问，再稍后重试；403/418/429 不一定代表 Cookie 已过期。
- 舆情池条件清理不可撤销；“一键彻底抹除系统大库”的影响范围更大，请勿混淆。
- 使用 OpenAI 或 DeepSeek 需要自行提供 API Key 并承担服务费用；AI 请求会向当前模型服务发送所选评论内容，但不会发送 Cookie、安全设置或本地路径。
- 安装包未进行 Authenticode 代码签名，Windows SmartScreen 可能提示未知发布者。请只从本仓库 Release 下载并核对 SHA-256。

### 验证与校验

- 后端 13 项测试、AI/更新/安全 smoke、Electron 脚本语法、TypeScript 和 Next production build 均通过。
- 使用真实加密 Cookie 的临时副本低数量验证了抖音、小红书和微博采集，未写入真实数据库。
- 隔离 userData 启动验证 FastAPI `0.2.3`、Next HTTP 200、自动退出及监听端口释放。

## v0.2.2 - 2026-07-13

### 新增与改进

- 舆情池新增“来源归属”，用于记录评论来自哪位作品作者或投稿人的内容；支持平台自动提取、项目主页覆盖、详情人工修改、筛选和导出。
- 系统通用设置新增“舆情池数据清理”，支持按项目、平台、日期、来源归属、情绪、风险、处理状态、周报状态和关键词组合预估及清理。
- Steam 商店评价新增时间范围和五语多选，支持简体中文、英语、日语、俄语和韩语，默认仅选择简体中文。
- AI 周报严格按页面勾选的章节生成，并将全局系统 Prompt 同时应用于每批分析和最终综合。
- AI 分析上下文新增来源归属统计，可比较不同投稿人的评论量、情绪、风险、标签、点赞和回复表现。
- 更新检查优先解析 GitHub 公开 Release 网页跳转，并增加 10 分钟缓存；未认证 API 配额耗尽时仍可检查最新版本。
- 更新错误提示可区分访问频率受限、网络超时、无公开 Release 和无效版本标签。

### 使用方法

1. 新用户运行 NSIS 安装程序，可选择安装目录并创建桌面或开始菜单快捷方式。
2. v0.2.1 用户可直接安装 v0.2.2。原项目、评论、周报、Cookie、安全设置和 AI 配置继续保存在当前 Windows 用户目录中，启动后会自动执行兼容迁移。
3. 在“游戏项目管理”中，可为平台主页填写来源归属覆盖值；留空时由支持的平台自动识别作品作者。
4. 在“舆情采集与评论池 → 本地平台采集”选择 Steam 后，可设置日期范围和评价语种；所选语种合计共用评论数量上限。
5. 在“系统通用设置 → 舆情池数据清理”中先组合筛选并点击“预估命中数”，核对数量、输入页面要求的确认文字后才可执行清理。
6. 在“周度通报自动化生成”中勾选需要的章节后再点击 AI 生成；不同来源归属的统计会随评论数据一起进入分析上下文。

### 注意事项

- 舆情池条件清理不可撤销。它只删除命中的评论及直接关联映射，保留项目、采集历史和既有周报正文；执行前仍建议导出备份。
- “一键彻底抹除系统大库”的范围更大，会删除项目、评论和周报，请勿与条件清理混淆。
- Steam 多语种采集结果会去重并共同受总量上限约束；日期按本机时区计算并包含起止当天。
- B站、抖音、小红书和微博采集仍需要用户保存有效 Cookie；本工具不会绕过验证码、登录风控或平台访问限制。
- 使用 OpenAI 或 DeepSeek 需要自行配置有效 API Key，并承担对应服务费用；使用 Ollama 前需启动本机或自定义地址的 Ollama 服务。
- AI 请求会将所选项目周期内的评论内容发送给当前激活的模型服务商，但不会发送 Cookie、API Key、安全设置或本地路径。
- 本版本安装包未进行 Authenticode 代码签名，Windows SmartScreen 可能提示“未知发布者”。请只从本仓库 Release 下载，并核对 SHA-256。

### 验证与校验

- 已验证旧 v1 数据库副本增量迁移，原有 2 个项目、130 条评论和加密配置保持兼容。
- 已完成后端、AI、更新、安全 smoke、TypeScript、production build、桌面/768px 界面和 NSIS 隔离安装验收。

## v0.2.0 - 2026-07-10

### 新增

- 更新仓库地址已内置，关于页无需再手动填写 GitHub 仓库。
- 玩家评论舆情池新增服务端翻页，每页最多 100 条，默认展示最新 100 条。
- 玩家评论舆情池新增自动化批量处理，可按当前筛选范围重算情绪、风险和自动标签并标记为已处理。
- 系统通用设置新增 OpenAI、DeepSeek、Ollama AI 配置档管理。
- 支持保存多个 API Key 配置档，并指定一个全局默认配置和模型。
- 支持读取服务商模型、自定义服务地址、全局系统 Prompt 和 Think/Reasoning 模式。
- 周度通报新增“AI 智能分析生成”，支持流式 Markdown、思考摘要展示、取消和超长评论自动分批汇总。

### 安全与兼容

- API Key、AI 系统 Prompt、Cookie 和锁屏安全配置均使用 Windows 系统加密能力保存在本机。
- AI 思考内容仅在生成过程中展示，不写入周报或数据库。
- 保留原规则模板周报功能，现有项目、评论、标签和周报数据无需迁移。

### 注意事项

- 使用 OpenAI 或 DeepSeek 前，需要用户自行配置有效 API Key 并承担对应服务费用。
- 使用 Ollama 前，需要本机或自定义地址的 Ollama 服务已启动并安装模型。
- 当前 Windows Portable EXE 未进行代码签名，可能触发 SmartScreen 提示。
